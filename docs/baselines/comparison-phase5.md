## F5.5 生产化路径建议

### 立即可做

1. **接入真实 LLM** — 替换 AI Mock Service 为 Claude API / MiniMax 调用
2. **数据库持久化** — 对话和快照从内存迁移到 PostgreSQL
3. **文件上传** — 实现附件上传存储和真实文档提取

### 短期改进

4. **认证升级** — X-User-Id → JWT + SSO 集成
5. **测试覆盖** — 补充单元测试和集成测试
6. **飞书集成** — Mock → 真实飞书 API
7. **前端组件提取** — 公共组件提取到 @filing/ui

### 中期路径

8. **Synapse 集成评估** — 基于调研结论，Synapse 平台需先补齐 S1-S5 能力
9. **性能优化** — 流式响应、缓存、数据库索引
10. **多租户** — 领域隔离、数据权限细化

### 关键决策点

- **是否依赖 Synapse**：调研结论建议独立实现，待 Synapse 补齐能力后再评估整合
- **AI 模型选择**：Claude 用于复杂推理（风险评估、摘要），轻量模型用于分类/提取
- **部署架构**：当前 4 版本可合并为单一渐进式应用，通过功能开关控制
